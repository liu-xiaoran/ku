后台管理系统文档

（本文默认起始地址为跟目录；即 index.html 所在位置）

启动准备

根目录内分为四个文件夹，正常页面开发只需改动 master 文件夹里的内容 和 server 文件夹内的json文件；
本地启动时，需要安装的全局环境程序包括，python，Visual Studio Tools，Visual C++ Build Tools，nodejs 等；node 在官网下载稳定版安装完毕后自动带有NPM；
建议将依赖下载地址改为淘宝的镜像，任意位置打开CMD，输入 npm i nrm -g完成后再nrm use taobao （此步可不做）
使用 NPM 在 ./master 文件夹下（package.json文件所在位置）安装全部依赖（在该文件夹内打开CMD，输入npm i）；安装完毕后 该目录下CMD输入 $ gulp serve 即可运行网页并监听文件修改自动刷新 即热刷新。

新建页面

angle框架的 模型-视图-控制 分开所以需要分别更改几个地方：

一,添加数据
在文件 ./server/sidebar-menu.json 中以JSON格式插入要添加的页面（由上往下渲染）；
如：添加 第一标题为运营管理第二标题为contractId内容按钮为contractId管理


格式为：

{
    "text": "运营管理",
    "heading": "true"
  },
  {
    "text": "contractId",
    "sref": "#",
    "icon": "icon-speedometer",
    "label": "label label-info", 
    "submenu": [
      {"text": "contractId管理",       "sref": "app.contract_info"}
    ]
  },

其中 "sref": "app.contract_info"为指向你的页面名称，如 内容管理 中的 站内信管理 content_sitemail 即写为 app.content_sitemail . （保存网页会自动刷新，即看到添加的新样式）

二，添加路由
在文件 ./master/js/modules/routes/routes.config.js 中添加页面所需依赖和页面指向地址；
如添加内容管理中的站内信管理的路由部分：

.state('app.content_sitemail', {
                // url中 “content_sitemail” 换成你的页面名称
              url: '/content_sitemail/:action',
              title: 'content_sitemail',
                // 地址中也换成你的页面名称.html
              templateUrl: helper.basepath('content_sitemail.html'),
              controllerAs: 'dash2',
                // 加载你需要的依赖（一般直接复制即可，看名称也大概猜到以下各依赖的功能，有特殊需求再添加其他）
              resolve: helper.resolveFor('flot-chart','flot-chart-plugins','angularFileUpload', 'filestyle','ngDialog',"ngTable","datatables")
          })

三，添加页面及添加控制
前两步做完后，网页内出现新的后台管理页面按钮，并且可点击，下面开始添加页面内容；
页面已经新建，所以书写页面和控制部分的方法，会写在以下修改页面内；

修改页面内容

1，jade文件
页面内容写在 ./master/jade/views/ 中，编辑的话找到对应文件打开，或新建对应名称的jade文件，格式为你的名称.jade，如 内容管理 中的 站内信管理 页面为 content_sitemail.jade
jade为前端模板引擎，保存时会自动编译成html，用jade写html相当于用markdown写文档，网上搜一下语法和注意事项即可上手；
建议先看一下jade语法 http://www.w3cplus.com/html/jade.html
注意事项：jade是通过语句前方的空格数来判定父子或兄弟关系的，注意格式对应；
jade模板看完语法，对应一个已有的页面，随便改改看看网页端的变化就会了；

angular的数据绑定用着很方便，下面列举页面内用到的；

2，控制器绑定
控制器的绑定：在页面需要数据交互的外部盒子上，添加控制器；如站内信管理页面内上层：
.box( ng-controller=" ContentSitemailController as index " )
在ContentSitemailController 处换成你的控制器的名称，对应于你控制器JS中命名的名称， index 也可以任意命名，下面数据绑定时 index.date 等也对于改掉即可。

3，ng-model
ng-model ：绑定了 HTML 表单元素到 scope 变量中 ；
即为将该元素与控制器中的变量绑定，实现动态同步；如输入手机号搜索功能中，
input.input-sm.form-control(type='text', placeholder='搜索手机号', ng-model='index.textPhoneSearch')
该元素绑定了 index.textPhoneSearch 在控制器中定义一个 this.textPhoneSearch 视图和控制器中同样名称的即可拿到或修改数据；一个变化后，另一个也同样变化。

4，{{index.data}}
{{index.data}} 绑定数据的显示，以两个大括号包住要显示的变量，写在内容里既可以动态变化；如上面绑定了手机号，如果要显示手机号，自命名{{index.textPhoneSearch}} 即可，不管是控制器中改变还是绑定的输入框中改变数据，该数据都会动态的显示；

5，ng-click
ng-click ：绑定点击事件，告诉页面点击该元素时需要执行的方法；
如：button.btn.btn-info(type='button', ng-click='index.reflash()') 刷新
表示点击该按钮是，执行 控制器中的对应名称为reflash（）函数；同样使用 index.reflash( ) 控制器中对应 this.reflash( )函数，index.的后面带括号为调用方法（函数），不带括号为绑定变量；注意：请勿让函数与变量重名；

调用时也可以传参，如：button.btn.btn-sm.btn-default(type='button',ng-click='index.phoneSearch(index.textPhoneSearch)') 搜索 直接将要调用的内容填在小括号即可；

6，ng-show/ng-hide
控制元素显示与隐藏：.allsite(ng-show='index.showaddstatus()') 可以绑定函数也可绑定变量，为真时显示或隐藏；

7，ng-change
表单元素变换时执行函数；一个函数可绑定多个元素；如：ng-change='index.nextEnabledchang()'

8，ng-repeat 循环渲染
对于数组中的元素，在表单头部定义即可循环出来 如：

tr(ng-repeat='data in index.channels',ng-class='success')
            //将返回数组文件中的对应数据绑定在下面，就可以动态显示
            td {{data.mesTitle}}
            td {{data.mailType}}
            td {{data.phone}}
            td {{data.mesType}}
            td {{data.requester}}
            td {{index.turnDate(data.createTime)}}
            td {{data.approver}}
            td {{index.turnDate(data.updateTime)}}
            td {{data.status}}

9，ng-table 表单绑定
在表单头部增加ng-table='index.tableParams5'即把表单与tableParams5插件绑定；tableParams5会在控制器中说明；

修改控制器

1，结构
控制器也就是控制该网页的JS文件，./master/js/custom 文件夹下，找到页面对应的文件夹和文件 .controller.js；
新增的话要新建文件夹，里面一般包含两个文件，即控制器 controller.js 和 模块 module.js ；
新增的module.js可以仿写以下：

(function() {
    'use strict';
    angular
        .module('app.content', []);
})();

其中的 app.content 替换成你新建的文件夹名称；并且在 ./master/js/app.module.js 文件中添加该模块引入字段；
控制器最开始定义模块和控制器名称并且引入依赖；和jade里面的控制器名要对应；

angular
        .module('app.content')
        .controller('ContentSitemailController', ContentSitemailController);

    ContentSitemailController.$inject = ['$http', '$scope', '$resource', '$location', 'ngDialog', 'ngTableParams', 'ngTableDataService', '$timeout'];

然后书写控制器，建议跟之前的文件一样，先把this定义为vm.

function ContentSitemailController($http, $scope, $resource, $location, ngDialog, ngTableParams, ngTableDataService, $timeout) {
        var vm = this;

        activate();
})

2，函数编写
在其中定义个activate();函数，你的所以JS就可以直接写在里面了；
变量直接写为 vm.变量名 = object ;
函数直接写为 vm.函数名 = funtion (){ };
注意与视图中对于同样的名称。即为绑定；
数据交互eg：

$http.post('接口地址', { 传出的交互数据 })
.success(function(res){
    此处填写成功后的函数
    res为返回的数据
})
.error(function(res){
    此处填写失败的函数
    res为返回的数据
})

.then(function(res){ 紧接着上一个函数执行完后执行的函数 }

3，tableParams5
表单插件，将表单交给该插件管理后，并将表单的数据交互写在插件内，既可以实行表格翻页和刷新；
eg:

vm.tableParams5 = new ngTableParams({
                page: 1,  // show first page 
                count: 10 // count per page  
            }, {
                total: 20, // length of data
                counts: [], // hide page counts control
                getData: function($defer, params) {
                    pageInfo.pageNo = params.url().page;
                    pageInfo.pageSize = 10;
                    $http
                        .post('/ZYManager/api/platforMail/list', {
                            data: data,
                            reqTime: '' + (new Date()).getTime(),
                            pf: "PC",
                            pageInfo: pageInfo
                        })
                        .success(function(response) {
        ..........
        })

接管表格后，在其他方法中，如果想刷新表格，执行：vm.tableParams5.reload();即可刷新表格数据；

添加弹窗模板

1，添加模板
点击按钮或者完成操作时弹出的样式，一般使用模板先将样式好，需要时在控制器JS中将其调用执行即可；
模板视图写在对应的页面的 jade文件 底部，如 已经写好的页面 contract_info.jade 或 content_sitemail.jade 等等 下方都有：

<!-- Templates -->
script(type="text/ng-template", id="dialogWithNestedConfirmDialogIds")
    .ngdialog-message
      h3.mt0 审核站内信
      .form-group(style='margin-bottom:20px')
        .col-md-12    
          label 内容
            span(style="color:red") *
          .row(style='margin:0 0 15px 0')
            textarea.input-sm.col-md-12(type='text',placeholder='请输入内容',ng—model='index.get_text',ng-change='index.nextEnabledchang()',rows=5) 
        .row(style='margin:0 0 15px 0')
          button.btn.btn-default.mr(type='button', ng-click="closeThisDialog('Cancel')") 取消
          button.btn.btn-danger.mr(type='button', ng-click="refuseds()",ng-disabled='!index.nextEnableds') 驳回
          button.btn.btn-primary(type='button', ng-click="confirm('Save')",ng-disabled='!index.nextEnableds') 通过

在控制器中添加以下模板调用代码，模板代码中template 与样式中 id 对应即可操作添加模板，（可复制之前写过的相似的再稍作改动）

ngDialog.openConfirm({
                        template: 'dialogWithNestedConfirmDialogIds',
                        className: 'ngdialog-theme-default',
                        scope: $scope,
                        ......

对照已有的写好的页面会发现，大多模板内写有preCloseCallback函数，这个是 点击取消时弹出的模板；不需要的可以删去。

如果该模板的 id 没有被定义，即是之前没有已存在的页面使用过该id模板，就会模板报错，可在 ./master/jade/views/ngdialog.jade 中添加该模板 id 即可；打开该文件，页面拉至下方，仿照上面已有的添加一个，修改掉id 即可；

2，添加模板控制
一般的模板会有默认的几个按钮，如 确认（confirm），返回（closeThisDialog）；仿照写好的页面，书写相应的动作后执行的函数即可，既成功后调动接口或失败后执行的函数；模板最后部位的那个函数，function（）{ 内填写所有执行后执行的函数 }；
如果需要自定义按钮，或更改按钮功能；直接先在模板样式对于jade文件中改动模板样式，再添加控制器即可；

模板控制器 当模板控制的规则你想自己写时或现有的不满足用途时，即可添加控制器，你也可以把以上默认的函数都写在控制器里；
如以上代码块中：一般在 scope: $scope, 后面，添加controller: ['$scope', function($scope) { 内容 }],
(注意后面的逗号，angular中很多元素以JS对象的格式存在，由于编译后才显示，所以浏览器中看不出具体错误原因，需要书写注意格式细节)

控制器定义添加完毕，在内容处直接先绑定 控制元素 再书写 控制代码 即可；格式为 $scope.绑定的名称 = function(){ 控制代码 } ；如以上默认确认返回按钮的也可重写
$scope.confirm = function(){ }

写为：$scope.refuseds = function(){ }
